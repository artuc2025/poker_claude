```

Прочитай следующие файлы проекта в таком порядке:
1. PROJECT_CONTEXT.md — общий контекст и текущий статус
2. ARCHITECTURE.md — структура и компоненты
3. RULES.md — code conventions
4. FEATURES.md — список фич и приоритеты
5. TASK_HISTORY.md — что уже сделано

Подтверди что понял контекст, скажи какая сейчас фаза и какой следующий таск.
```

```
После выполнения:
1. Покажи все созданные / изменённые файлы
2. Обнови TASK_HISTORY.md с записью о выполненном таске
3. Обнови статус в FEATURES.md (⬜ → ✅)
4. Обнови PROJECT_CONTEXT.md (Current Task / Completed Tasks)

```

## Tips для эффективного vibe coding

1. **Один таск = один чат.** Не пытайся сделать 5 тасков в одном чате — агент потеряет фокус.

2. **Всегда прикрепляй все 5 файлов** в начале нового чата. Это контекст агента.

3. **После каждого чата** — забирай обновлённые PROJECT_CONTEXT.md и TASK_HISTORY.md. Это твоя "память".

4. **Приоритет тасков**: сначала все 🔴 HIGH в фазе, потом 🟡 MED, потом переходи к следующей фазе.

5. **Не пропускай фазы.** Phase 1 → Phase 2 → ... Фундамент важен.

6. **Если агент предлагает альтернативное решение** — пусть запишет его в TASK_HISTORY.md в "Decisions made".

7. **Code review prompt**: после каждых 5 тасков попроси агента проверить весь код на соответствие RULES.md.
